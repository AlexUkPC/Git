Git Config

In Git Bash

git config --global user.name "Alex Rogna"

Setup name

git config --global user.email alexrogna@yahoo.com

Setup email

code

Test if Visual Studio Code can start from gitbash

git config --global core.editor "code --wait"

Setup Visual Studio Code as core editor

git config --global -e

open the config file for Git in Visual Studio Code

git config --global core.autocrlf true

Auto delelete or add end of lines so it works on any OS(windows true/linux macOS input

Setup Visual Studio Code as diff tool for Git

git config --global diff.tool vscode

git config --global difftool.vscode.cmd "code --wait --diff $LOCAL $REMOTE"

git config --global -e

add $LOCAL $REMOTE on the last row if they are not there

Git Commands

mkdir Moon

create a directory name Moon

cd Moon

enter the directory Moon

git init

initialize git in directory

ls

list files in directory

ls -a

list all files in directory

echo helllo > file1.txt

create a file named file1.txt with text hello inside it

git status

check the status of files in directory

git add file1.txt file2.txt

add file1.txt and file2.txt in staging area

git add *.txt

add all txt files in staging area

git add .

add all files in staging area

echo world >> file1.txt

append word world in file1.txt

git commit

commit the changes with a long description

git commit -m "Initial commit"

commit the changes with a short description

rm file2.txt

remove file2.txt from working directory

git ls-files

list files in staging area

git rm file2.txt

remove files from both working and staging area in the same time

echo logs/ > .gitignore

create a file .gitignore with a list of files and folder git should ignore when commit

git rm --cached -r bin/

remove a directory from staging area if we commited and after we added it to .gitignore

=== premade .gitignore file for diffrent programming language : github.com/github/gitignore ===

git status -s

short version of satus

git diff --staged

show what are the differences that we have in the staging area that are going in the next commit

git diff

show what are the differences that we have in the working area that are not yet in staged

!!!next 2 lines works only if Visual Studio Code is setup as diff tool before - see Git Config section for more details

git difftool --staged

show in Visual Studio Code what are the differences that we have in the staging area that are going in the next commit

git difftool

show in Visual Studio Code what are the differences that we have in the working area that are not yet staged

git log

show history

git log --oneline

short history

git log --oneline --reverse

short history in reverse

git show 5954c95

show the commit with id 5954c95

git show HEAD~1

show the commit with number 1 from HEAD

git show HEAD~1 : .gitignore

show only what was commited for the file after the colon

git ls-tree HEAD~1

show commit with number 1 from HEAD as tree

git restore --staged file1.js

remove file1.js from staging area

git restore file1.js

restore file in working directory(undo local changes)

git restore .

restore all files in working directory except for the new created ones which are not yet tracked so git doesn't know about them

git clean -fd

restore all files in working directory

git restore --source=HEAD~1 file1.js

resore file1.js to a previous version

touch file2.html

create file2.html

git branch login

create login branch

git checkout login

switch to login branch

git merge login

merge login branch to main

GitHub

git remote add origin http://github.com/...

select location on github to upload

git push -u origin main

upload to github the main branch

git pull

download from github

git clone http://github.com/...

clone a repository from github
